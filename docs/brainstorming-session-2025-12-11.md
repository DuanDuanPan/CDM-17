## Session Overview

**Topic:** 将协同研发平台从传统项目-任务流水线切换为“脑图驱动的协同研发模式”，要求具备XMind级脑图特性与专业UI/UX和灵活交互
**Goals:** 1) 定义脑图核心能力与交互细节；2) 设计脑图驱动的研发流程映射（需求/任务/数据流）；3) 形成支撑PRD的材料（需求要点、场景、功能、优先级、验收标准）

### Context Guidance

### Technique Selection
**Approach:** AI-Recommended Techniques
**Analysis Context:** 需求为脑图驱动协同研发 + PRD支撑，需兼顾专业UI/UX与灵活交互。

**Recommended Techniques:**
- **Mind Mapping (structured):** 直接对应目标形态，便于梳理“脑图驱动研发”核心对象、节点类型、属性、交互。
- **Constraint Mapping (deep):** 识别并可视化研发流程约束（权限、版本、同步、合规、国产化、离线等），指导功能与技术方案。
- **Decision Tree Mapping (structured):** 设计“从脑图到研发流转”的决策路径（节点→需求/任务→交付），明确分支策略与落地规则。

**AI Rationale:** 先用 Mind Mapping 描述目标形态与信息模型，再用 Constraint Mapping 暴露约束与机会，最后用 Decision Tree Mapping 将脑图动作映射到研发流程、权限与交付路径，形成可落地的 PRD 结构。


## Technique Execution Results

### Mind Mapping — 信息模型（深挖）
- 节点类型：产品结构、需求、任务、输入模板、输出模板、知识、专业工具/App、控制（审批/点检流程）。
- 字段模型：按类型提供基础字段（标题、描述、状态、优先级、负责人、截止、标签、引用、版本等），并支持动态扩展（自定义字段/继承）。
- 关系：父子、前驱/后继；MBSE关系（需求实现/验证/追溯等），支持跨图引用与镜像。
- 状态机：草稿 → 进行中 → 完成 → 归档（含暂停/恢复）；支持锁定或并发编辑控制。
- 视图：以树为主，结合节点类型需要可切换逻辑/时间/泳道/演示等形式（根据后续设计细化）。

### Mind Mapping — 驱动执行（深挖）
- 触发与映射：手动启动任务；自动触发遵循前驱/后继计划逻辑（FS/SS/FF/SF），仅针对任务节点；支持定期执行任务；可配置触发条件满足即触发。
- 自动化与提醒：状态提醒（参考 project 状态控制），与计划驱动逻辑联动。
- 前驱后继：基于 FS/SS/FF/SF 阻塞/并行规则执行；异常/阻塞解除后再触发后续。
- 外部系统：暂无指定，对接规划待定。

### Mind Mapping — 数据流传递（深挖）
- 传递对象：需求与数据可在上下游传递。
- 上下游动作：上游可执行下发；下游可接收/驳回；变更可触发通知。
- 变更通知：支持变更提醒（需设计通知范围、方式、节流策略）。
- 待补充：跨图同步粒度、字段映射、版本合并/冲突处理、血缘可视化。

### Mind Mapping — 协同与权限（深挖）
- 协同状态：可实时查看多人在线编辑状态；支持锁定/解锁；冲突检测。
- 权限控制：节点/分支级访问与编辑控制（待细化字段级/附件级）。
- 互动：评论与回复，支持@提及。
- 审计/追溯：待补充（操作日志、合规要求）。

### Mind Mapping — 交互与UX（深挖）
- 布局：支持自由布局、树、逻辑等多种视图，可切换。
- 快捷键：体系参考 XMind；需可自定义或至少可查看快捷键帮助。
- 操作：拖拽、批量操作、快速创建（/命令、双击、粘贴成树、AI生成初稿）。
- 关系线：多样关系线样式（含依赖/数据流/跳连），可标注标签和方向。
- 下钻：当大图过大时，选中某节点可下钻进入其子图，专注局部并保持与主图的同步回链。

### Mind Mapping — 绩效管理（深挖）
- 分值与分摊：支持任务/节点绩效分值设定；支持比例分摊给个人/组。
- 提醒与激励：支持提醒/激励机制（规则待定）。
- 统计与排名：按个人、组、部门多维统计与排名；支持看板指标展示。
- 导出：支持导出绩效相关报表。

### Constraint Mapping — 约束与假设
- 安全/保密：暂不纳入本阶段（需后续补充）。
- 权限/协同：冲突由人为合并（非自动）；锁/解锁与协同已在前述设计。
- 离线/版本：暂不支持离线编辑；节点多版本并存，支持差异对比与时间线。
- 性能/可用性：大图≥1k节点需流畅；实时协同延迟目标 <200ms；高并发通知需节流。
- 集成与数据主权：暂不考虑（专业工具/国产化/数据驻留后续评审）。
- 变更与通知：站内信推送；需定义范围与频率节流规则。


### Decision Tree Mapping — 脑图到执行路径
- 映射对象：仅任务节点生成/同步任务；触发时机为状态变更（含创建/提交）。不同节点类型需配置必填字段映射表（标题/负责人/截止/优先级/标签/引用等）。
- 依赖阻塞：支持 FS/SS/FF/SF 关系（结束-开始、结束-结束、开始-开始、开始-结束）作为阻塞/并行规则。
- 版本与可见性：未保存版本仅本人可见，支持自动保存；发布版本决定推送，版本切换需通知。
- 审批/控制：交付物审批通过后方可下发；任务完成可选审批，可选择“任务+交付物一起审批”；审批通过后按机制动态推演下一个待执行任务并发送相关消息。
- 通知：状态变更、版本切换、驳回、完成等事件发送站内信（需定义节流/汇总策略）。


## Idea Organization & Priorities

### 高优先级 (Must)
- 脑图核心：自由/树/逻辑布局；快捷键（XMind风格）；拖拽、批量、快速创建；多关系线（依赖/数据流）。
 - 大图下钻：支持节点下钻到子图，便于超大图的分层浏览与编辑。
- 信息模型：节点类型（产品结构、需求、任务、模板、知识、工具、控制）；多版本+时间线；状态机草稿/进行/完成/归档；MBSE关系。
- 驱动执行：任务节点映射任务；状态变更触发；FS/SS/FF/SF 阻塞规则；定期任务；站内信通知。
- 协同与权限：实时在线指针；锁定/解锁；冲突检测（人为合并）；节点/分支权限；评论@；审批链（交付物/任务）。
- 性能目标：≥1k 节点流畅；实时协同 <200ms；通知节流。
- 版本与可见性：未保存版本仅本人可见；自动保存；版本切换通知。

### 中优先级 (Should)
- 数据流传递：上下游下发/接收/驳回；变更通知；（待定）跨图同步粒度、血缘可视化。
- 模板与导入导出：结构模板库；XMind/Markdown/CSV 导入；PNG/HTML/Markdown 导出。
- 绩效管理：分值设定与分摊；多维度统计与排名；看板指标；报表导出；提醒/激励规则。

### 低优先级 (Could / 后续)
- 集成与国产化/数据主权：专业工具/国产化/数据驻留后续评审。

### Party Mode Notes — 权限与审计细则
- 密级：用户密级需 ≥ 资源密级方可见；低密级自动隐藏敏感字段/附件元数据；可申请临时提升（审批）。
- 节点授权：节点可绑定多人，逐人配置可见/可编辑/可下载/可审批；默认继承父分支，可局部覆盖。
- 水印与审计：在线预览水印（用户名+时间+IP）；预览/下载均写审计；审计可按人/节点/密级过滤导出。
- 日志保留：建议保留≥1年，可按策略裁剪/脱敏。


### Party Mode Notes — 通知节流共识
- 触发事件：创建/状态变更/版本切换/驳回/完成/审批结果。
- 信道：站内信为主；首版不支持 IM webhook。
- 节流与汇总：同节点同事件5分钟内去重；批量汇总窗口默认5分钟（可配置）。
- 优先级：阻塞/驳回/审批结果为高优先，实时送达；普通提醒走汇总。
- 订阅：按用户/角色/节点/分支订阅；当前不提供静音时长/范围。
- 视图：通知收件箱可按节点/事件类型/时间过滤，已读/未读管理。


### Party Mode Notes — 性能基线共识
- 场景：单图≥1k节点（含关系线/附件元数据），多人协同。
- 并发：压测档位 50 人并发编辑。
- 客户端性能：初始加载<2s；缩放/拖拽/折叠 95% 响应<100ms；关系线绘制不卡顿。
- 协同延迟：端到端<200ms；批量操作可容忍<400ms。
- 数据策略：懒加载分片（按层级/视口）；大附件延迟加载；关系线按视图裁剪。
- 压测脚本：加载/展开/批量粘贴/拖拽/折叠/版本切换；记录 P95/P99。
- 监测：前端 FPS/内存/事件耗时；后端队列时延、冲突率、通知推送耗时。
- 降级：不允许自动降级（需在满功能下满足指标）。
- 移动端：首版不要求移动端同等指标。

### Advanced Elicitation — First Principles 项目简介草稿
1. 根本目标：将协同研发从“项目-任务-汇报”流水线转为“脑图驱动的实时协同与执行”，让结构化思考与任务落地同源。
2. 本质对象：脑图节点 = 需求/任务/模板/知识/控制；节点携带版本、状态、密级和关系（父子、依赖、数据流）。
3. 关键约束：1k+ 节点无降级性能；实时协同<200ms；密级/水印/审计；离线暂不支持；通知 5 分钟去重+汇总。
4. 最小可行能力：
   - 多布局脑图 + 快捷键/批量/下钻；
   - 任务映射与 FS/SS/FF/SF 依赖；
   - 审批后推演下一任务；
   - 站内信节流通知；
   - 自动版本与时间线。
5. 差异化：密级字段/附件权限 + 水印审计；跨图血缘/数据流可视化；无降级性能基线；审批驱动的执行编排。
6. 成功度量：
   - 50 并发、1k 节点下 P95 交互<100ms；
   - 审批→下一任务自动分派命中率；
   - 告警/通知重复率下降（5 分钟去重）；
   - 权限违规审计零漏记。


### Advanced Elicitation — Shark Tank Pitch 简介
- 痛点：传统“项目→任务→汇报”割裂，需求/架构/执行分层，协同慢、通知噪音大、权限粗粒度、性能在大图场景劣化。
- 解法：脑图即研发的单一真相源（SoT）。多布局脑图承载需求/任务/模板/知识/控制；FS/SS/FF/SF 依赖与审批编排驱动执行；站内信节流；密级+水印+审计保障安全。
- 差异化：
  1) 密级/字段/附件级权限 + 动态水印审计；
  2) 跨图血缘与数据流可视化；
  3) 无降级性能基线（1k+节点、50并发）；
  4) 审批后自动推演下一任务的执行编排。
- 商业价值：
  - 研发周期缩短：审批→下发自动化，减少手工分派/跟踪；
  - 质量与合规：密级与审计降低违规风险；
  - 可视透明：血缘/通知节流/版本时间线减少沟通成本；
  - 可扩展：模板库与AI分解降低建图和入门成本。
- 需求场景：多团队并发、涉密/分级访问、需要跨专业集成的系统工程/航天/通信/大型软件。
- 成功信号：上线3个月内 P95 交互<100ms（1k节点）；审批→自动分派命中率>90%；通知重复率下降>60%；权限违规零漏记；NPS>30。
